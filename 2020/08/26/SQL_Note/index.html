SQL常用命令即库操作

在终端中输入mysql -u[用户名] -p后弹出输入密码行，输入后进入mysql
查看当前库使用show databases;，注意在mysql中所有语句以;结尾
进入库使用use [库名];
查看库中表使用show tables;，在库中查看其它库表使用show tables from [库名];
查看表结构使用desc [表名];

库及表管理操作

事务控制

由一组sql语句组成一个执行单元，类似原语，要么全部执行，要么不执行
这是人为进行设计的单元，为了防止发生错误造成两个本应该同步执行的两条语句只执行了一半，后面提及的insert、update和delete均为隐式事务语句，而两条以上的语句合并就为显示事务，但需要先执行set autocommit = 0将自动提交禁用后再开启事务

类似OS中对资源的分配，在多个事务对同一个表单数据进行操作时会发生不可预知的错误，为了避免此类并发问题，引入与PV锁操作类似的隔离操作
在终端中输入select @@transaction_isolation;，可查看隔离级别，一般为READ级别，尽量避免脏读、不可重复读和幻读现象的出现

视图

为了满足某些特定需求根据普通表生成一个临时表，并不持续存在，且只存sql逻辑，类似将多个语句封装为一个函数，需要时调用即可

使用时可将视图名作为单独的表看待

存储过程

于上面的视图类似，将执行语句封装成一个方法，只不过视图的作用是生成一个表对象，而存储过程是执行语句的方法集合体

对于参数的操作为主要三种：

  • in：将某值作为参数传入存储过程in [变量名] [类型](长度)
  • out：作为返回值
  • inout：既可作为输入值也可作为返回值

常用数据类型

下面列出常见数据类型，括号内为字节数

  • 整形：Tinyint(1) Smallint(2) Mediumint(3) int(4) Bigint(8) (unsigned)类型后加为无符号
  • 浮点型：float(4) double(8)
  • 定点型：dec(M,D)(M+2) 有效取值范围由M和D决定，M为字节长度，D为小数长度
  • 字符型：char(字符长度至255) varchar(可变字符至65535) enum(枚举) text blob
  • 日期型：date(4) datetime(8) timestamp(4) time(3) year(1)

    查询

    基础查询

    查询某张表的单个或多个字段使用select [字段名1],[字段名2] from [表名];
    查询所有字段使用select * from [表名];
    用select方法也可查询常量、表达式和函数，如： 可对查询结果起别名，使用select [字段名1] as [别名1],[字段名2] as [别名2] from [表名];
    用空格替代as效果一样，不过有关键字冲突的风险，select [字段名1] [别名1],[字段名2] [别名2] from [表名];
    对于一字段中重复的对象，可用distinct去重，select distinct [字段名] from [表名]；
    在sql中‘+’符号只能进行数值运算，若对象不是数值则尝试转换后运算，转换失败则视为0
    若需要两字符拼接，则使用concat函数，select concat('字符1','字符2','字符3') from [表名];
    若查询须拼接的字段已为字符型，参数可直接使用字段名，select concat([字段名1],[字段名2],[字段名3]) from [表名];

    需要注意的是在做拼接或运算时遇到值为null的参数，结果一定为null，在使用中根据需要可进行一些变化
    使用ifnull()函数可以在遇到null值时输出其他参数，不为null时输出原值，ifnull([字段名],[输出值]);

条件查询

sql中除了字段，也可按条件查询

类似于编程语言中的条件表达式，sql中的条件和逻辑运算符为：

  • > 大于
  • < 小于
  • <> 或 != 不等于
  • <= 大于等于
  • >= 小于等于
  • and 与
  • or 或
  • not 非

实例SELECT last_name,department_id FROM employees WHERE salary>12000;
sql中支持模糊条件的搜索

  • like 可模糊查询指定字符，同时支持通配符%(表示0或任意多个字符)和_(表示任意单个字符)，在指定字符和通配符冲突时使用转义字符\，查询字段中第三位为a的对象：select [字段名] from [表名] where [字段名] like '__a%';
  • between and 描述字段是否置于指定上限和下限之间，等价于>= 和 <=，and左右的两个值包含临界值且顺序不能颠倒，select [字段名] from [表名] where [字段名] between [上限] and [下限];
  • in 描述字段中是否有指定字符，等价于=，in中的指定字符不支持通配符，select [字段名] from [表名] where [字段名] in('字符1','字符2');
  • is null 由于=和!=不能判断null值，故采用is null或is not null，select [字段名] from [表名] where [字段名] is null;

    sql中存在一个名为安全等于的符号<=>，它可以直接判断是否等于null或普通数值，但可读性较差
    select [字段名] from [表名] where [字段名] <=> [null或普通数值];

排序查询

对于输出的结果可进行排序处理，同样类似于条件查询，在语句后加入排序查询的字段即可实现

分组查询

利用group by语句配合分组函数进行分组，需要注意的是分组依据的字段必须同时存在于select查询列表中，支持用逗号隔开多个查询

若分组同时还需进行筛选操作，若在先筛选后分组则使用where语句，若先分组后筛选则需要使用having语句
select [分组函数],[字段] from [表名] group by [字段] having [筛选条件];
having需要位于order by前面

连接查询92/99（笛卡尔乘积）

同python类似，可在两张表中分别查询两个字段，在where语句中可用.号进行识别分类
select [字段1],[字段2] from [表1],[表2] where [表1].[字段1] = [表2].[字段2];

等值/非等值连接

同上述示例相同，在where中对两表的两个字段进行等值或非等值的比较，可在from后面为表起别名，并在select和where中使用别名，注意一旦使用了别名，就不能使用原名，与别名的顺序无关，且连接的表数量不限

自连接

对于需要在一张表中遍历两次的情况，可为表起两个别名，相当于将两张一样的表连接到一起
select [字段1] from [表1] A,[表1] B where A.[字段1] = B.[字段1];

99语法

上述语法为sql92规范下的写法，在sql99推出后语法类型略有变化
连接两表使用的是join语句具体类型为

其中的连接类型主要为：

  • inner(内连接)，和等值连接效果相同，相当于根据条件求交集
  • left outer(左外连接),right outer(右外连接)，用左外连接左边是主表，右外则反，外连接查询时是以主表为底，从表与其一致的数据，故总长(即行数)与主表一致，存在则为相应值，不存在则为null，将where条件设为查询key is null时则为取反效果，一般将主要查询字段的所在表设为主表
  • full outer(全外连接)，相当与左右外连接+内连接的效果，类似于两表合集的效果，取反与上述相同两表条件用or连接
  • cross(交叉连接)，即将两表进行笛卡尔乘积，表1的数据对于若干个表2数据

子查询

类似与函数嵌套，可在select from where having exists后添加小括号嵌套执行

标量子查询

适用于select where having后使用，输出结果为一行一列

行/列子查询

多行子查询一般配合in notin(列表任意一个) any some(列表某一个) all(列表所有)配合，即将上述的逻辑运算符换为相应语句即可，在跟子查询实现效果，用在from后面时相当于将子查询作为一个新表，同时新表必须起别名,exisits关注子查询中是否存在查询结果

联合查询

使用union语句使用，主要功能为将多条查询语句合并为一个结果
select [字段1] from [表1] union (all) select [字段2] from [表2]
适用场景大致可以理解为，连接查询是需要有连接条件才能使用，当两个表之间没有连接条件但又需要输出为一个表时就使用union语句

需要注意使用该语句时要保证两个查询的字段（即列数）和顺序是一致的
union语句是会自动去重，当不需要去重时使用union all语句即可

插入、修改、删除

插入操作

插入使用insert语句，语法为insert into [表名](字段1,字段2) values(新值1,新值2),(新值1,新值2);
values中新值的数量要和字段一一对应，且值类型要符合该字段的类型，新值根据表的设计可以为null值
第二种语法为insert into [表名] set [字段]=[值],[字段]=[值];
第一种语法可以插入多行，且可以嵌套子查询，即将查询的结果视为新值插入，故需要忽略values语句

修改操作

使用update语句，语法为update [表名] set [字段]=[值],[字段]=[值] where [条件];
因为使用的是字段定位，若不加where限定条件就会将所有行的数值全部修改，故添加where条件限定修改的行数
修改多表记录时需要先将表连接后再进行修改

删除操作

单表删除语句为delete from [表名] where [筛选条件]，delete操作是将符合该条件的数据整行删除
多表删除语句为delete [别名1],[别名2] from [表1] [别名1] [连接类型] join [表2] [别名2] on [连接条件] where [筛选条件]
第二种删除语法为truncate table [表名]，由于不能加where，所以为整表删除，使用delete删除主键的断点保留，truncate删除主键重置

函数

单行函数

常见函数：

  • lenght(str)，返回字节长度
  • concat(str,str)，参数为若干个字符串，返回拼接好的一个字符串
  • upper(str),lower(str)，将字符转换为大/小写并返回
  • subter(str,pos,len)，截取字符串，涉及函数重载，返回截取好的字符串，注意在sql中起始位置为1，而不是其他语言常见的0
  • instr(str, substr)，返回字符串中字串第一次出现的位置
  • trim('str' from str)，去除字符串中头尾存在的指定字符
  • lpad(str,len,'str')，左填充指定字符至指定长度，右填充为rpad()

    数学函数：

  • round(num，len)，将数值四舍五入，保留指定长度小数并返回，truncate(num,len)，仅截取指定小数
  • ceil(num)，将数值向上取整，floor(num)为向下取整
  • mod(num, num)，取模操作
    还有一些流程控制函数如

    分组函数

    将一组数据进行处理后输出一个值，类型严格，常见的函数有：
  • sum(字段)，求和
  • avg(字段)，求平均值
  • min(字段)，最低
  • max(字段)，最高
  • count(字段)，个数 #count()和count(1)都可以查行数，但1的效率比\较高
    可配合distinct做去重操作

    创建函数

    与其他语言中的函数含义相同
